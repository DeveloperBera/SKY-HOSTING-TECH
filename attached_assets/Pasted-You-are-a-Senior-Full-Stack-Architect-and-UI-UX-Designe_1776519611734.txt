## 3. Detailed Feature Specifications

### A. Admin Panel & API Key Management
- **UI:** A dedicated route `/admin`. Show a card with "Master API Key" (can be rotated). Display total deployments, active WebSocket connections, and server load.
- **API Endpoint:** `POST /api/v1/keys/generate` -> Returns a scoped key (e.g., `sk_live_xyz`).
- **Functionality:** The API Key must be valid in the `Authorization: Bearer sk_live_xyz` header. Implement middleware that checks the key against the User model.

### B. The Deployment Engine (The "Seconds" Promise)
- **File Upload:** Drag-and-drop folder handling. Zip it on client, send to `/api/deploy/upload`. Backend unzips to `/tmp/sky-[id]`.
- **Git Clone:** `POST /api/deploy/git` with `{ repoUrl: "...", branch: "main" }`.
- **Build Process Simulation (Important for Demo):**
  Since we cannot run actual insecure containers in a standard Node environment, create a **MockBuilder** service that emits progressive `stdout` logs over Socket.io with realistic delays (npm install -> build -> success). *Add a comment in code: "// REPLACE WITH DOCKERODE FOR PRODUCTION"*.
- **URL Generation:** After "deployment", generate a subdomain: `https://[project-name]-[random-hash].skyhosting.app` (simulate via local `/preview/:id` route for frontend demo).

### C. UI/UX: "So Cool" Factor
- **Background:** Deep space blue gradient (`#0B0F19` to `#1A2333`) with a rotating wireframe globe (use `react-three-fiber` or a pure CSS animated mesh).
- **Cards:** Blurred glass background with 1px white border and 10% opacity.
- **The "Deploy Bar":** A floating input bar at the top center of the dashboard. It accepts a Git URL or a folder drop. Upon submission, a vertical timeline animation plays showing: `Cloning в†’ Detecting Runtime (Python 3.11) в†’ Building в†’ Live`.
- **Live URL Button:** Once done, a neon green "LIVE" badge appears. Clicking it opens the deployed site in a new tab (even if it's just a static echo server for demo).

### D. API Control (The "Everything" Feature)
Provide the exact OpenAPI 3.0 YAML spec embedded in the `/docs` page. Implement these exact endpoints:
- `GET /api/v1/projects` - List projects.
- `POST /api/v1/deploy` - Trigger new deployment (body includes env vars JSON).
- `DELETE /api/v1/deployments/:id` - Takedown.
- `GET /api/v1/logs/:deploymentId` - Returns log history.
